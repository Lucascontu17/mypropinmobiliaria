[v0.3.0] - ACTOR MANAGEMENT & DIGITAL FILE IMPLEMENTATION
Fecha: 2026-03-31
Rama: dev (Testing)

Gestión Integral de Propietarios e Inquilinos (CRUD):
- Creación de las vistas `/propietarios` e `/inquilinos` basadas en DataTables nativas de Shadcn (tipografía Inter en tablas/form).
- Integración de formularios (React Hook Form) modal-centric para Alta/Edición.

Control de Integridad de Datos (Zod) y El Búnker:
- Esquematización estricta: DNI requerido, Email nativo, y Teléfono bajo formato E.164 (soporte Twilio futuro).
- Prohibición de montos de liquidación o balance en negativo.
- Estrategia *Stale-while-revalidate* implementada con SWR para refrescos síncronos optimizados del listado al invocar la mypropAPI v1.6.0.
- Aislamiento en cliente: inyección forzosa del `inmobiliaria_id` dentro del scope local de cada request CRUD, suprimiéndolo deliberadamente del endpoint UI.
- Placeholders `dni_url` y `contrato_url` resueltos estáticamente como string (pre-Storage).

[v0.2.0] - STATELESS AUTH & RBAC IMPLEMENTATION
Fecha: 2026-03-31
Rama: dev (Testing)

Seguridad Stateless (Clerk):
- Implementación del Provider centralizado con soporte en español y Appearance minimalista base (Renta520).
- Diseño TypeSafe de los roles en runtime: 'superadmin' | 'admin' | 'vendedor'.
- Middlewares de Frontend: `<ProtectedRoute>` verifica el `inmobiliaria_id` y valida jerarquías por acceso en UI sin tocar la base de datos local ni latencias API.

Pantallas de Autenticación:
- Modo B2B (Centered Modal Window) para SignUp / SignIn enrutados, garantizando aislamiento de la SPA principal.

Reglas del Master Filter Refinadas:
- `useInmobiliaria()` expone `hasPermission(allowedRoles[])` posibilitando la ocultación dinámica de los botones y de los submenúes laterales de navegación garantizando el principio "Zero Leaks".

[v0.1.0] - REPOSITORY INITIALIZATION
Fecha: 2026-03-31
Rama: dev (Testing / Staging)

Stack Técnico:
- Runtime: Bun 1.3.10
- Bundler: Vite 6.4.1
- Framework: React 19.2.4 + TypeScript 5.7.2
- Estilos: Tailwind CSS v4.2.2 (plugin Vite nativo)
- UI Library: Shadcn/ui (estilo new-york, CSS variables)
- API Client: Eden Treaty 1.2.0 → mypropAPI v1.6.0
- Auth: Clerk React v5 (publicMetadata multi-tenant)
- Iconografía: Lucide React v1.7.0
- Validación: Zod v4.3.6

Identidad Visual (Luxury Minimalist):
- Paleta: Renta520 (10 tonos teal/esmeralda, #f5f8f8 → #102324)
- Título: Plus Jakarta Sans (semibold, tracking-tight)
- Formularios/Tablas: Inter (400-600)
- Detalles Luxury: Playfair Display (italic, serif)

Arquitectura Multi-tenant:
- Master Filter: inmobiliaria_id extraído de Clerk publicMetadata
- Hook dedicado: useInmobiliaria() para inyectar el filtro en todas las queries
- Aislamiento de datos: Toda query al Búnker requiere inmobiliaria_id obligatorio

Branching Strategy:
- main: Rama de Producción. Solo merges vía PR certificado tras QA en Staging.
- dev: Rama de trabajo activa para desarrollo de etapas 1-7 y despliegue a Staging.
- Commits directos en main: PROHIBIDOS.

Estructura del Panel:
- Layout: Sidebar colapsable + Topbar + Content area
- Dashboard: KPIs de resumen (Propiedades, Inquilinos, Cobranza, Ocupación)
- Rutas planificadas: /propiedades, /inquilinos, /cobranzas, /configuracion
